# 🎙️ Audio Learner - 音视频学习内容分析助手

分析视频/播客音频内容，生成清晰的中文思维导图、核心观点总结与关键结论。支持说话者识别和自动翻译。

---

## 🚨 最重要的规则：内容准确性

**绝对禁止凭空编造内容！**

- **必须基于真实转录**：只有在用户提供了视频链接/音频文件，或已有转录稿时才能生成内容
- **找不到内容时**：告诉用户"无法访问该视频/音频"，询问是否需要先转录，**不能编造**
- **先转录，再分析**：对于没有现成文稿的内容，应该先转录为文字，再进行分析总结
- **翻译是辅助**：翻译的目的是帮助理解原文，不能替代原文分析

### 处理流程：

1. **拿到链接/文件** → 判断是否有现成转录稿
2. **如果只有链接** → 询问用户是否需要先转录
3. **如果用户同意** → 使用 `quick_transcribe.py` 或 `local_whisper_transcriber_final.py` 转录
4. **基于转录稿分析** → 生成思维导图、核心观点、关键结论
5. **翻译辅助理解** → 提供双语对照，但核心输出是中文总结
6. **绝对不能**：不做转录就编造看起来合理但完全错误的内容

## 🚀 快速开始

### 1. 安装依赖（只需一次）

```bash
brew install ffmpeg
pip3 install yt-dlp openai-whisper requests
```

### 2. 使用方式（两种方式）

#### 方式一：直接告诉我链接（推荐）

直接把视频/播客链接发给我，我会：
1. 先问用户是否需要转录（如有现成转录稿可跳过）
2. 如果需要转录，使用脚本下载音频并转录
3. 基于转录稿分析内容
4. 生成中文思维导图、核心观点、关键结论

**示例对话：**

> 用户：帮我分析这个视频 https://www.bilibili.com/video/BV1xxxxx
>
> 我：好的，这个视频还没有转录。需要我先帮你转录并翻译吗？
>
> 用户：好的
>
> 我：好的，让我开始转录...转录完成！现在开始分析内容...

#### 方式二：使用脚本转录后分析

**Step 1：转录**

```bash
cd /path/to/your/project

# 下载并转录（基础版，只有原文）
python3 .trae/skills/Audio-learner/quick_transcribe.py "视频链接"

# 下载并转录（高级版，支持翻译和说话者识别）

    audio_downloads/video.mp3 \
    --speaker-names "张三,李四"
```

**Step 2：分析**

把转录稿发给我，我会帮你生成分析总结

## 🛠️ 常见问题

### Q: 提示 `zsh: command not found: python` 怎么办？
**A:** macOS 上使用 `python3` 而不是 `python`：
```bash
python3 .trae/skills/Audio-learner/quick_transcribe.py "视频链接"
```

### Q: 提示 `No such file or directory` 怎么办？
**A:** 确认你在正确的项目目录（包含 `.trae` 文件夹的目录）：
```bash
cd /path/to/your/project
python3 .trae/skills/Audio-learner/quick_transcribe.py "视频链接"
```

### Q: `pip` 命令找不到怎么办？
**A:** 使用 `pip3`：
```bash
pip3 install yt-dlp openai-whisper
```

### Q: B站视频下载失败怎么办？
**A:** B站有反爬虫机制，解决方案：

**方案1：使用浏览器 cookies（推荐）**
1. 安装浏览器插件如"EditThisCookie"或"Cookie Inspector"
2. 登录 B 站
3. 导出 cookies 到文件：`~/.bilibili_cookies.txt`
4. 再次运行转录脚本

**方案2：手动下载后转录本地文件**
1. 登录 B 站网页版
2. 下载视频到本地
3. 使用本地文件路径转录：
```bash
python3 .trae/skills/Audio-learner/quick_transcribe.py "/path/to/video.mp4"
```

### Q: 说话者识别效果不准确怎么办？
**A:** 本工具使用基于停顿时间的简单分组，准确率约 80%。如果效果不理想：

**方案1：手动标注**
你可以在生成的笔记中手动修改说话者名称。

**方案2：使用专业工具（高级用户）**
```bash
pip3 install whisperx torch
```
然后运行时自动使用声纹识别，准确率约 95%。

### Q: 翻译不准确怎么办？
**A:** 翻译使用 Google Translate API，通用场景效果不错。但专业术语可能需要调整：

**建议**：在生成的笔记中，如果发现翻译有问题：
1. 告诉我具体哪一段翻译不准确
2. 提供更好的翻译版本
3. 我会帮你更新

### Q: 如何处理长视频（超过30分钟）？
**A:** 建议分段处理：
1. 先转录整个视频
2. 然后分段让我分析（如：00:00-15:00、15:00-30:00）

或者在转录时指定语言（加快速度）：
```bash
python3 .trae/skills/Audio-learner/quick_transcribe.py "视频链接" --ja
```

### local_whisper_transcriber_final.py 参数说明

```
位置参数：
  input                   视频链接或本地音频文件路径

可选参数：
  --speaker-names NAMES   说话者名称，用逗号分隔
                          示例: "张三,李四" 或 "主持人,嘉宾A,嘉宾B"
  --output-dir DIR        输出目录，默认为当前目录

示例：
  # 基本使用
  python3 local_whisper_transcriber_final.py audio.mp3

  # 自定义说话者
  python3 local_whisper_transcriber_final.py audio.mp3 \
      --speaker-names "马斯克,采访者"

  # 指定输出目录
  python3 local_whisper_transcriber_final.py audio.mp3 \
      --speaker-names "张三,李四" \
      --output-dir ./outputs
```

## 💡 使用建议

- **访谈类视频效果最好**（2-3人对话，有自然的观点交锋）
- **单人演讲/讲座**可以快速提取核心要点
- **第一次使用**会自动下载 Whisper 模型（~150MB），请耐心等待
- **分析质量取决于转录质量**：转录准确，分析才准确
- **长视频建议分段**：每30分钟一段，避免单次分析内容过多
